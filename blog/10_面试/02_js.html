1.null 和 undefined

  • 一般未指定指向对象的指针会被默认为Null类型，因为一个值为Null的变量其实际类型为object，因为系统会将其视为一个保存对象的变量，只不过还未初始化。
  • Java理解就是：null像在Java里一样，被当成一个对象，对于 Java 来说不可能存在一个“不存在”的成员（不存在的话直接就编译失败了），所以只用 null 来表示语义上的空值。
  • 在JavaScript中，将一个变量赋值为undefined或null，几乎没区别。
  • 出现undefined原因：
    • JavaScript的最初版本没有包括错误处理机制，发生数据类型不匹配时，往往是自动转换类型或者默默地失败
    • Brendan Eich（开发者）觉得，如果null自动转为0，很不容易发现错误。所以整了个undefined

相似点

  • undefined和null在if语句中，都会被自动转为false，相等运算符甚至直接报告两者相等。

 if (!undefined) {
   console.log('undefined is false');// undefined is false
 }

 if (!null){ 
   console.log('null is false');// null is false
 }

 console.log(undefined == null); //true
 console.log(undefined === null); //false

 console.log(Number(undefined)); //NaN
 console.log(5 + undefined); //NaN

 console.log(Number(null)); //0
 console.log(5 + null); //5

2.Object.is与"===","=="区别

异同

ES5 比较两个值是否相等，只有两个运算符：相等运算符（==）和严格相等运算符（===）。前者在比较时对于不同数据类型的值会进行类型转换；而后者不会转换，且结果更容易预测，并且因为没有隐式转换，全等比较的运行会更快。因此建议使用严格相等运算符(===)进行值的比较。
Object.is 用来比较两个值是否严格相等，与严格比较运算符（===）的行为基本一致。解决了严格相等运算符（===） NaN 不等于自身，以及 +0 等于 -0的问题。

严格相等运算符 ===

  1. 不同类型的值比较 如果比较的两个值的类型不同，直接返回false。
1 === "1"          // false
true === "true"    // false

  1. 非数值类型比较 (值相同就返回 true，值不同就返回 false。)
null === null        // true
null === undefined   // false

1 === 1 //true
// (特殊) 0不分正负
0 === -0//true
// (特殊) NaN 本身不全等于任何值。
NaN === NaN     //false
NaN !== NaN     //true

{} === {}                             // false
[] === []                             // false
(function () {} === function () {})   // false

相等运算符==

相等运算符(==)用来比较相同类型的数据以及复合类型数据时，与严格相等运算符(===)完全一样。比较不同类型的数据时，相等运算符会先将数据进行类型转换，然后再用严格相等运算符比较。

  1. 原始类型的值会转换成数值再进行比较。转换规则是使用 Number() 转换。
 1 == true             // true   等同于 1 === Number(true)
 0 == false            // true   等同于 0 === Number(false)
 'true' == true        // false  等同于 Number('true') === Number(true)
 '' == 0               // true   等同于 Number('') === 0
 '' == false           // true   等同于 0 === 0
 '\n  123  \t' == 123  // true  因为字符串转为数字时，省略前置和后置的空格

 false == undefined     // false
 0 == null              // false
 0 == undefined         // false
 NaN == undefined       // false
 undefined == null      // true

[1] == 1   // true  等同于 Number([1]) == 1

Object.is(value1, value2)

严格比较运算符（===）的行为基本一致 不同之处只有两个：一是+0不等于-0，二是NaN等于自身。

 +0 === -0            //true

 Object.is(+0, -0)    // false
 Object.is(NaN, NaN)  // true

对于不兼容 ES6 的浏览器，可以在ES5 环境下通过下面的代码，部署Object.is。

Object.defineProperty(Object, 'is', {
  value: function(x, y) {
    if (x === y) {
      // 针对+0 不等于 -0的情况
      return x !== 0 || 1 / x === 1 / y;
    }
    // 针对NaN的情况
    return x !== x && y !== y;
  },
  configurable: true,
  enumerable: false,
  writable: true
});

3.箭头函数中的this指向

this作为关键字，它的作用就是引用，并且它通常只写在函数内部就是函数体内

定义

  • 普通函数： this与其定义位置无关，谁调用这个函数，this就指向谁。
  • 箭头函数： this跟调用者无关，定义位置所在的作用域的this（函数作用域）是谁，箭头函数的this就指向谁

示例：

普通函数(全局中)：

 window.name = 'windowName'
 function demo1() {
   console.log(this.name)
 }
 let a = {}
 a.name = 'a'
 a.speak = demo1

 // 直接执行demo1函数，虽然看上去没有其他东西去调用demo，实际上是window在调用,所以this指向window
 demo1() // windowName
 // 第二次我们用a去调用，则this指向a
 a.speak() // a
