防抖

  • 当一个动作连续触发，只执行最后一次
  • 👐：好比游戏回城按键，点击回城8秒后，就会执行回城事件（回到泉水），但是如果在这8秒内，再次点击回城按键，回城就会被打断，重新计算8秒,回城这个操作只执行了最后一次
  • 一般使用场景

搜索框搜索 用户输入完成之后，再发送请求，不是每次输入或者改变都发送请求
登录、发短信等按钮 防止点击太快，以至于发送多次请求

案例(三种方式)

0
防抖
<template>
  <div>
    <xl-button @click="add1" type="primary">+1(没有防抖)</xl-button>
    <xl-button @click="add2" type="primary">+1(有防抖)</xl-button>
    <xl-button @click="add3" type="primary">+1(封装防抖)</xl-button>
    <xl-button @click="add4" type="primary">+1(lodash防抖)</xl-button>
    {{num}}
  </div>
</template>

<script setup>
  import { ref } from 'vue';
  let num = ref(0)
  
  // 没有防抖
  function add1(){
    num.value++
  }


  /**
   * 第一次点击时设置了定时器 如果时间内没有再次点击改按钮 则定时器没有被清除 时间到了就会执行
   * 如果再次点击按钮，上一次的定时器会被取消，然后重新设置一个一秒钟的定时器 时间到了就会执行
   */
  let timer = null;//time用来控制事件的触发
  function add2(){
    if(timer){  //判断是否已经存在定时器
      clearTimeout(timer)  //存在 则清除定时器
    }
    // 设置定时器 或者重新设置定时器
    timer = setTimeout(() => { 
      num.value++  
    }, 1000);
  }


  // 使用自己封装的防抖
  const add3 = debounce(add1,1000)
  // 防抖封装
    /*
     * fn : 回调函数  传进来的是正真的业务逻辑 
     * delay : 延时时间
     */ 
  function debounce(func, delay) {
    let timer;// 设置定时器标识
    
    // 难点 返回事件绑定函数
    return function() {
      let context = this;// func指定this
    	let args = arguments;// func参数
      if(timer){
    	  clearTimeout(timer);// 先清除定时器
      }
    	//设置定时器
    	timer = setTimeout(() => {
    		// 调用函数 call apply绑定this后会自动执行
    		func.apply(context, args);
    	}, delay);
    }
  }


  // 使用lodashjs
  // import lodash from 'lodash'  //我这里已经全局引入 所以不再引入了
  const add4 = lodash.debounce(add1,1000)

  // vue2 使用防抖
  // methods: {
  //   add4: lodash.debounce(function() {
  //     // 业务代码
  //   }, 400)
  // }
</script>

节流

  • 当一个动作连续触发，会按照一定频率执行，不会触发一次就执行一次
  • 👐：好比王者荣耀的技能按钮，连续点击技能按钮，并不能连续的放出技能，只能等冷却好了之后才能再次放出技能。
  • 一般使用场景

懒加载时要监听计算滚动条的位置，但不必每次滑动都触发，可以降低计算的频率，而不必去浪费 CPU 资源；
鼠标连续不断地触发某事件（如点击），只在单位时间内只触发一次；

案例

0
防抖
<template>
  <div>
    <xl-button @click="add1" type="primary">+1(没有节流)</xl-button>
    <xl-button @click="add2" type="primary">+1(封装节流)</xl-button>
    <xl-button @click="add3" type="primary">+1(封装节流2)</xl-button>
    <xl-button @click="add4" type="primary">+1(lodash节流)</xl-button>
    {{num}}
  </div>
</template>

<script setup>
  import { ref } from 'vue';
  let num = ref(0)
  
  // 没有节流
  function add1(){
    num.value++
  }


  // 时间戳版
  function throttle(func, wait) {
      let previous = 0; //记录上一次的开始时间
      return function() {
        let now = Date.now(); //获取当前时间
        let context = this;
        let args = arguments;
        //当前时间-上一次开始时间 与等待时间作对比
        // 如果大于等待时间就可以执行函数了  并且记录当前的时间（previous）
        // 如果小于等待时间 ，则还不能执行（技能还没冷却好） 不执行（放不出技能）
        if (now - previous > wait) { 
          func.apply(context, args);
          previous = now;
        }
      }
  }

  // 定时器版
function throttle2(func, wait) {
    let timeout; //设置定时器标识
    return function() {
      let context = this;
      let args = arguments;
      // 标识是空的才能执行下面的方法
      if (!timeout) { 
        // 第一次是空的 设置了一个定时器
        // 定时器时间到后  再次将标识设置为空 并执行函数（放出技能）
        // timeout 处于非空状态时 ，就是 冷却还没完成
        timeout = setTimeout(() => {
          timeout = null;
          func.apply(context, args)
        }, wait)
      }
    }
}

  // 使用自己封装的防抖（时间戳版）
  const add2 = throttle(add1,1000)


  // 使用自己封装的防抖（定时器版）
  const add3 = throttle2(add1,1000)



  // 使用lodashjs
  // import lodash from 'lodash'  //我这里已经全局引入 所以不再引入了
  const add4 = lodash.throttle(add1,1000)

  // vue2 使用防抖
  // methods: {
  //   add4: lodash.throttle(function() {
  //     // 业务代码
  //   }, 400)
  // }
</script>
