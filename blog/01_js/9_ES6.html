1.var let const区别

关键字变量提升块级作用域重复声明同名变量重新赋值
var√×√√
let×√×√
const×√××

描述

  • var
  1. var声明的变量存在变量提升，即变量可以在声明之前调用，值为undefined ​
  2. 一个变量可多次声明，后面的声明会覆盖前面的声明 ​
  3. 在函数中使用var声明变量的时候，该变量是局部的作用域只在函数内部，而如果在函数外部使用 var，该变量是全局的
  • let
  1. 不存在变量提升，let声明变量前，该变量不能使用。就是 let 声明存在暂时性死区 ​
  2. let命令所在的代码块内有效，在块级作用域内有效，作用域只是在花括号里面 ​
  3. let不允许在相同作用域中重复声明，注意是相同作用域，不同作用域有重复声明不会报错
  • const
  1. const声明一个只读的常量，声明后，值就不能改变 ​
  2. let和const在同一作用域不允许重复声明变量const声明一个只读的常量。一旦声明，常量的值就不能改变，但对于对象和数据这种 引用类型，内存地址不能修改，可以修改里面的值。 ​
  3. let和const不存在变量提升，即它们所声明的变量一定要在声明后使用，否则报错

2.proxy

只做简到不能再简的简介，详细用法请查阅MDN和阮一峰ES6

理解

Proxy 可以理解成，在目标对象之前架设一层“拦截”，外界对该对象的访问，都必须先通过这层拦截，因此提供了一种机制，可以对外界的访问进行过滤和改写。Proxy 这个词的原意是代理，用在这里表示由它来“代理”某些操作，可以译为“代理器”。

  • 语法
let p = new Proxy(target, handler);

function setProxy() {
  // 给水果创建一个代理 代理之后 只要是请求水果里面的属性都必须经过代理 
  fruit = new Proxy(fruit, {
    get(target, key) {
      let result = target[key];

      // 给价格限制范围
      if (key === "price" &&  value >100) {
        throw Error("可不能太贵了");
      }
      console.log(`已设置价格为${value}`);
      return Reflect.set(target, key, value);
    },
  });
  isProxy.value = true
  proxyNum.value++
  console.log("代理成功");
}

</script>

3.Promise(★)

解析

1、Promise 是异步编程的一种解决方案，主要用于异步计算，支持链式调用，可以解决回调地狱 的问题，自己身上有all、reject、resolve、race 等方法，原型上有then、catch等方法。
2、可以将异步操作队列化，按照期望的顺序执行，返回符合预期的结果，可以在对象之间传递和操作 promise，帮助我们处理队列
3、promise 有三个状态：pending[待定]初始状态，fulfilled[实现]操作成功，rejected[被否决]操作失败
4、Promise 对象状态改变：从pending变为fulfilled和从pending变为rejected。只要这两种情况发生，状态就凝固了，不会再变了
5、如果不设置回调函数，Promise内部抛出的错误，不会反应到外部，但是写了then 和 catch ，会被then的第二个参数 或 catch所捕获

  • promise 的 then 为什么可以支持链式调用

promise 的then会返回一个新的 promise 对象，能保证 then 方 可以进行链式调用

所以，从表面上看，Promise只是能够简化层层回调的写法.
而实质上，Promise的精髓是“状态”，用维护状态、传递状态的方式来使得回调函数能够及时调用
它比传递callback函数要简单、灵活的多。所以使用Promise的正确场景是这样的：

catch的用法

上面案例可以知道Promise对象除了then方法，还有一个catch方法，它是做什么用的呢？其实它和then的第二个参数一样，用来指定reject的回调，用法是这样：

methodName().then(function(data){
    console.log('resolve');
    console.log(data); //这里data就是methodName里面resolve(data)的data
})
.catch(function(reason){
    console.log('rejected');
    console.log(reason); //同理 //这里reason就是methodName里面reject(reason)的reason
});

all、race的用法

  • all方法的效果实际上是「谁跑的慢，以谁为准执行回调」
  • Promise的all方法提供了并行执行异步操作的能力，并且在所有异步操作执行完后才执行回调。
  • 使用场景，打开资源多的网页时，先加载处需要的资源，再进行页面初始化。
//  Promise.all([异步方法1,异步方法2,异步方法3])
Promise.all([asyncMethod1(), asyncMethod2(), asyncMethod3()])
.then(results => {
    console.log(results); //结果以为数组输出
}).catch(err => {
    console.log(err);
});
// 输出 ['asyncMethod1结果', 'asyncMethod2结果', 'asyncMethod3结果']

4.ES6模块和CommonJS

对比

  1. import是ES6中的语法标准也是用来加载模块文件的，import函数可以读取并执行一个JavaScript文件，然后返回该模块的export命令指定输出的代码。export与export default均可用于导出常量、函数、文件、模块，export可以有多个，export default只能有一个。
  2. require 定义模块：module变量代表当前模块，它的exports属性是对外的接口。通过exports可以将模块从模块中导出，其他文件加载该模块实际上就是读取module.exports变量，他们可以是变量、函数、对象等。在node中如果用exports进行导出的话系统会系统帮您转成module.exports的，只是导出需要定义导出名。

require与import的区别

  1. require是CommonJS规范的模块化语法，import是ECMAScript 6规范的模块化语法；
  2. require是运行时加载，import是编译时加载；
  3. require可以写在代码的任意位置，import只能写在文件的最顶端且不可在条件语句或函数作用域中使用；
  4. require通过module.exports导出的值就不能再变化，import通过export导出的值可以改变；
  5. require通过module.exports导出的是exports对象，import通过export导出是指定输出的代码；
  6. require运行时才引入模块的属性所以性能相对较低，import编译时引入模块的属性所所以性能稍高。

作用：

  • 彻底解决了js文件依赖混乱和全局变量污染的问题。

ES6和commonJS的出现

v8引擎发布之后,出现了Node,js这门语言应用到了后端，作为一个后端语言连木块话技术都没有，是很难发展壮大的，于是社区就开始了讨论各种实现模块化的方法，最后commonJS也就诞生了,因此Node也应用了commonJS的标准和理念来实现模块化。
但是commonJS和AMD和CMD都是属于社区讨论出来的方案。直到ES6的发布，ECMA官方才正式发布ES6模块化标准。
总结为：commonJS只适用于Node,而ES6模块化支持Node和浏览器端。

// 直接导出整句导出，导入变量名要跟导出的模块变量名一一对应否则报错，导入时用{}包裹的变量名
export const name = 'wenl'
export const age = 18

// import.js
// 名称要跟导出的模块名称一一对应
import { name, age } from './export.js' 
console.log(`我叫${name2},今年${age}岁`) // 我叫wenl,今年18岁

======================================分割线=================================================
// export.js
// 也可以先定义变量名，再直接导出变量名。 (推荐这种 一眼就能看到导出的有什么参数)
const name2 = 'wenl2'
const age2 = 19

export {
  name2,
  age2
}

// import.js
import { name2 , age2 } from './export.js' 
console.log(`我叫${name2},今年${age}岁`) // 我叫wenl2,今年19岁

======================================分割线=================================================
// export.js
export const name = 'wenl'
export const age = 18

// import.js 
// as对变量起别名：
import { name as myName, age as myAge } from './export.js' 
console.log(`我叫${myName},今年${myAge}岁`) // 我叫wenl,今年18岁

======================================分割线（注意点）=================================================
// export.js
const name = 'wenl'
const age = 18
export {
  name,age
}
// import.js
// 导入变量(非引用类型)是只读的，不可修改。
import { name, age } from './export.js'
age = 20  //Uncaught TypeError: Assignment to constant variable. 未捕获的类型错误：赋值给常量变量。
console.log(`我叫${name},今年${age}岁`) // 不会打印 上一步已经报错了

======================================分割线（注意点）=================================================
// export.js
// 导出引用类型（对象，数组）可修改值
const student = {
  name:'wenl',
  age: 18
}
const arr = [1,2,3]
export {
  student,
  arr
}
// import.js
// 导入引用类型（对象，数组）可修改值
import { student, arr } from './export.js'
student.name = 'WENL'
student.age = 20
console.log(student.name, student.age) // WENL 20
console.log(arr[0])  // 1
arr[0] = 10
console.log(arr[0])  // 10

2.导出导入函数

// export.js
export function add(a, b) {
  console.log(a + b)
}
// import.js
import { add } from './export.js'
add(1, 2) // 3

======================================分割线=================================================
// export.js
// 导出也可以起别名(add as myAdd)
function add(a, b) {
  console.log(a + b)
}
function sub(a, b) {
  console.log(a - b)
}
export {
  add as myAdd,
  sub
}
// import.js
// 导入起别名（sub as mySub）
import { myAdd,sub as mySub } from './export.js'
myAdd(1, 2) //3
mySub(2, 1) // 1

3.整体导入

在导入的时候用*号指定一个对象，同时用 as 给这个对象起别名，那么导出的全部变量都是这个对象的某个属性。

// export.js
function add(a,b){
    console.log(a+b);
}
function sub(a,b){
    console.log(a-b);
}
let arr = [1,2,3];
export {add , sub , arr}

// import.js
import * as obj from './export.js'
obj.add(2,3);
obj.sub(6,3);
console.log(obj.arr);

4.默认导出

export default 指定默认输出，就是导出的时候为导出一个匿名函数，而在导入的时候就可以指定随便起一个名字都能实现导入。这样就不需要知道原模块输出的函数名，大大方便使用。且导入的时候不再需要{}包裹。要注意的是，一个js文件（模块）只能存在一个export default ：

// 导出函数
// export.js
export default function(a,b){
  console.log(a+b);
}
// 或
function add(a, b) {
  console.log(a + b)
}
export default add

// import.js
import fn from './export.js'
fn(1,2);

======================================分割线=================================================
// 导出对象
// export.js
export default {
  add(a, b) {
    console.log(a + b)
  },
  name: 'wenl'
}
// 或
function add(a, b) {
  console.log(a + b)
}
const name = 'wenl'
export default {
  add,
  name
}

// import.js
import obj from './export.js'
obj.add(1, 2) // 3
console.log(obj.name) //wenl

======================================分割线=================================================
// export.js
export default function(name,age){
    this.name = name;
    this.age = age;
}

// import.js
import Person from './export.js'
var zhangsan = new Person('zhangsan',19);
console.log(zhangsan);

5. 复合写法

如果在一个模块中，导入一个模块后又导出同一个模块，可以采用复合写法：

import {add} from './add.js'
export {add}
// 或
export {add} from './add.js'
