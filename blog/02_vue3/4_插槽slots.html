官网

理解

  • 插槽组件可以当成一个复用（通用）模板组件。如多个组件的布局使用相似模板，只是具体内容不同，那么我们可以使用这种插槽方式封装成一个通用组件，在其他组件使用的时候只需要传对应的内容到对应名字的插槽即可，不需要将该模板在每个组件重新写一遍，减少代码冗余，大大提高开发效率。
  • 人话：定义了一个杯子组件，想喝牛奶就装牛奶，想喝茶就装茶。不用每一种饮料都重新定义一个一模一样的杯子。

具名插槽

  • 一个组件可以多个插槽 但是需要给插槽起名字
  • 理解：给鸳鸯锅两边标注好那一边是辣的 那一边是装清汤的 辣的汤底就放辣的那一边 清汤的就放清汤的那一边
    父组件（v-slot 语法糖=> #）
<template>
  <div>
    <Children>
      <!-- 使用具名插槽A -->
      <template #A>
        <p>A插槽插入内容</p>
      </template>
      <!-- 使用具名插槽B -->
      <template v-slot:B>
        <p>B插槽插入内容</p>
        <p>{{data1}}</p>
      </template>
    </Children>
  </div>
</template>

<script setup>
  import { ref } from "vue";
  import  Children  from "./components/Children.vue";  
  const data1 = ref('B插槽数据123')
</script>

作用域插槽

  • 父组件提供内容，在子组件中展示。
  • 子组件提供内容（数据），在父组件中展示。

官方解析：组件有编译作用域---父级模板里的所有内容都是在父级作用域中编译的，子模板里的所有内容都是在子作用域中编译的。
所以作用于插槽延伸了子组件的作用域

  • 理解 我定义了一个杯子，我需要告诉使用的人，我这个杯子，只能装300mL，这时我们就需要用 vue2 👉 solt="data1" slot-scope 💨或者 vue3 👉 v-slot:A="data1" 语法糖 #A="data1"来接收子组件上通过v-bind绑定的值。
  • 使用场景例子：在使用一些表格组件时，通常我们还需要对表格进行编辑或者删除，此时我们可以通过作用域插槽获取组件传回来的索引以及内容，这样就可以对表格的对应数据进行编辑。

父组件

<template>
  <div>
    <Children>
      <!-- 这里就体现了 拓展了子组件的作用域 -->
      <template #A="data">
        <p>插槽A(杯子A传来的容量){{data.size}}</p>
      </template>
    </Children>
  </div>
</template>

<script setup>
  import  Children  from "./components/Children.vue";  
</script>
